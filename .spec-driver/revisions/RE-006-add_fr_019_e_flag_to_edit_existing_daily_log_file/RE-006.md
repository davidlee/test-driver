## 1. Context
- **Why**: User feedback during DE-003. The `-e`/`--edit` flag opens today's
  existing log file directly in the editor for free-form editing, distinct
  from editor mode which composes a *new* entry.
- **Source Specs**: PROD-001
- **Origin**: IMPR-001 (backlog improvement)

## 2. Related Artefacts
- **Backlog**: IMPR-001

## 3. Summary of Changes

### FR-019 (new) — Edit existing daily log
> When `-e` flag is provided, system MUST open today's log file in the
> detected editor. If no file exists for today, system MUST exit with a
> message. On save, the file is updated in place (no append logic).

Added to PROD-001 capability `read-mode` (alongside FR-012, FR-013, FR-017).

## 4. Consequences
- **Positive**: Completes the "interact with today's log" surface — read,
  pager, path, and now edit.
- **Negative**: Reuses editor detection from `internal/editor` but bypasses
  the append/timestamp path — needs clear separation.
