### Input mode detection

```
args present after flags?  в†’  inline mode (FR-005/006)
stdin is not a TTY?        в†’  pipe mode (FR-007)
otherwise                  в†’  editor mode (FR-008)
```

Priority: explicit args > pipe > editor. urfave/cli handles `--` natively.

### Config schema (`im.toml`)

```toml
# all keys optional вЂ” defaults shown
log_dir = "~/log"
editor = ""                    # falls back to $EDITOR, then $VISUAL
editor_timestamp = "start"     # "start" | "end"
timestamp_rounding = "adaptive" # "adaptive" | "round10"
```

Resolution: TOML file loaded at startup, values used as flag defaults.
Env vars (`IM_` prefix) supported via urfave/cli's `EnvVars` on flags.

## 5. Design Decisions & Trade-offs

- **DEC-001 вЂ” Flat internal packages**: One file per package for now. Split
  only when a package exceeds ~200 lines. Keeps navigation trivial.

- **DEC-002 вЂ” [urfave/cli](https://github.com/urfave/cli) for CLI**:
  Lightweight, single-dep CLI framework. Handles flags, `--` separator,
  help generation, and can source defaults from env/config. One dep
  covers both CLI and config concerns вЂ” no need for a separate config lib.

- **DEC-003 вЂ” golangci-lint with aggressive rules**: Config copied from
  `vice` project. Revive, gosec, gocritic, cyclop (max 10), errcheck with
  type-assertion and blank checks, no naked returns. Zero-warning policy.

- **DEC-004 вЂ” [mockery](https://github.com/vektra/mockery) for mocks**:
  If interfaces need mocking in tests, use mockery for generation. Not
  added as a dep until actually needed.
