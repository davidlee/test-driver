## 1. Summary & Context
- **Product Spec**: [PROD-001](../../product/PROD-001/PROD-001.md) — Interstitial Mindfulness Logger
- **Implementation Plan**: [IP-005](./IP-005.md)

## 2. Motivation
Two remaining feature gaps in the logging surface:

1. **Template engine** (FR-014, FR-015): Daily files are currently created with
   a hardcoded format (`# Day, DD Month YYYY`). Users want customisable daily
   file structure — frontmatter, custom headings, dynamic variables. Go
   `text/template` with a bundled default template that reproduces current
   output.

2. **12h time format** (FR-016): Timestamp subheadings are currently 24h only.
   Some users prefer `2:30 PM` over `14:30`.

Both are config-driven formatting changes sharing the config and logfile code
surfaces.

## 3. Scope & Objectives
- **Primary Outcomes**:
  - Daily file creation uses a Go `text/template` from `~/.config/im/template.md`
    or a bundled default (FR-014)
  - Template variables: `created_at`, `updated_at`, `id` (7-char nanoID),
    `title` (from configurable format string), `date` (FR-015)
  - When frontmatter contains `updated_at`, it is patched on each append (FR-015)
  - `time_format` config key: `"24h"` (default) or `"12h"` (FR-016)
  - Bundled default template produces output identical to current hardcoded format
- **Operational Constraints**: One new dependency: `go-nanoid/v2` for FR-015.
- **Dependencies**: None (DE-003, DE-004 completed).

## 4. Out of Scope
- FR-008 (zed `--wait`) — blocked upstream
- Template inheritance / partials — v1 is single-file template
- Date-based template selection — all days use same template

## 5. Approach Overview
- **`internal/template`** (new package): Load template from config path or
  embed default. Render with variable context. Bundled default reproduces
  current `dateHeading` output.
- **`internal/logfile/appender.go`**: On new file, delegate to template engine
  instead of `dateHeading()`. On append when frontmatter has `updated_at`,
  patch it.
- **`internal/logfile/timestamp.go`**: Accept `TimeFormat` config; format
  display time as `15:04` or `3:04 PM`.
- **`internal/config`**: Add `TimeFormat`, `TitleFormat` fields + validation.
- **`cmd/im/main.go`**: Pass config through to appender (already does).

## 6. Verification Strategy
- **FR-014**: Default template produces identical output to current hardcoded
  format. Custom template renders correctly. Missing template falls back to
  default.
- **FR-015**: Variables populated correctly. `updated_at` patched on append
  when present. `id` is 7-char nanoID. `title` rendered from format string.
  `updated_at` left alone when absent.
- **FR-016**: 24h produces `## 14:30`, 12h produces `## 2:30 PM`. Config
  validation rejects invalid values.
- **Acceptance**: `just check` exits 0, all existing tests still pass.

## 7. Risks & Mitigations
- **Risk**: Template rendering adds latency — *Likelihood*: low — *Impact*:
  low — *Mitigation*: Only on new-file path, template is small.
- **Risk**: Frontmatter patching corrupts existing files — *Likelihood*:
  medium — *Impact*: high — *Mitigation*: Only patch `updated_at` line;
  test with various frontmatter shapes; skip when field absent.
- **Risk**: nanoID dependency — *Likelihood*: low — *Impact*: low —
  *Mitigation*: Well-maintained, small dependency.
