# DR-005 – Template engine and 12h time format

## 1. Executive Summary
- **Delta**: [DE-005](./DE-005.md)
- **Status**: draft
- **Synopsis**: Replace hardcoded daily file creation with Go `text/template`,
  add frontmatter `updated_at` patching, and support 12h time format for
  subheadings.

## 2. Problem & Constraints
- **Current Behaviour**: Daily files created with hardcoded
  `# Monday, 2 January 2006` heading. Subheadings always 24h (`## 14:30`).
  No frontmatter, no customisation.
- **Drivers**: FR-014 (template), FR-015 (dynamic variables + updated_at),
  FR-016 (12h time).
- **Constraints**: Bundled default must produce identical output to current
  format. No new config keys required for basic operation (zero-config
  principle). One new dependency allowed (go-nanoid).
- **Out of Scope**: Template inheritance, date-based template selection,
  configurable template path (fixed at `~/.config/im/template.md`).

## 3. Architecture Intent
- **Target Outcomes**:
  - New files rendered from Go `text/template` (FR-014)
  - Template variables: `created_at`, `updated_at`, `id`, `title`, `date` (FR-015)
  - `updated_at` in frontmatter patched on each append (FR-015)
  - `time_format` config: `"24h"` (default) or `"12h"` (FR-016)
- **Guiding Principles**: The template engine is a new package boundary.
  The appender delegates to it for new-file creation but retains ownership of
  the append and timestamp logic. Frontmatter patching is a contained concern
  within the appender — it reads the file anyway for `ParseLastTimestamp`.

## 6. Code Sketches

### 6.1 Template Package — `internal/template/template.go`

```go
package template

import (
  "bytes"
  "embed"
  "os"
  "text/template"
  "time"
)

//go:embed default.md
var defaultTmpl string

// Context holds the variables available to daily file templates.
type Context struct {
  Date      string // "Monday, 2 January 2006"
  CreatedAt string // "2006-01-02 15:04"
  UpdatedAt string // "2006-01-02 15:04"
  ID        string // 7-char nanoID
  Title     string // rendered from title_format
}

// Render loads the template from path (falling back to the bundled default)
// and renders it with the given context.
func Render(templatePath string, ctx Context) (string, error) {
  tmplText := defaultTmpl
  if data, err := os.ReadFile(templatePath); err == nil {
    tmplText = string(data)
  }

  t, err := template.New("daily").Parse(tmplText)
  if err != nil {
    // Fall back to default on parse error.
    t, _ = template.New("daily").Parse(defaultTmpl)
  }

  var buf bytes.Buffer
  if err := t.Execute(&buf, ctx); err != nil {
    return "", err
  }
  return buf.String(), nil
}
```

**`internal/template/default.md`** (embedded):
```
# {{ .Date }}
```

This produces `# Monday, 2 January 2006\n` — identical to current
`dateHeading()`. No frontmatter, no variables beyond `Date`.

A custom template might look like:
```
---
created_at: {{ .CreatedAt }}
updated_at: {{ .UpdatedAt }}
id: {{ .ID }}
title: {{ .Title }}
---

# {{ .Title }}
```

### 6.2 Title Format Rendering

`title_format` in config is itself a Go template rendered with a `date`
variable:

```go
func renderTitle(titleFormat string, t time.Time) string {
  if titleFormat == "" {
    return ""
  }
  tmpl, err := template.New("title").Parse(titleFormat)
  if err != nil {
    return titleFormat // literal fallback
  }
  var buf bytes.Buffer
  tmpl.Execute(&buf, map[string]string{
    "date": t.Format("Monday, 2 January 2006"),
  })
  return buf.String()
}
```

Config example: `title_format = "Log — {{ .date }}"` → `"Log — Monday, 2 January 2006"`.

Note: uses `{{ .date }}` (dot-prefixed) since the context is a map.

### 6.5 Frontmatter `updated_at` Patching

When the existing file has `updated_at:` in its frontmatter, the appender
must patch it before appending. Strategy: line-level regex replacement.

```go
var updatedAtRe = regexp.MustCompile(`(?m)^updated_at:\s*.*$`)

func (a *Appender) hasUpdatedAt(content []byte) bool {
  // Only look in frontmatter (between first --- and second ---)
  return updatedAtRe.Match(frontmatter(content))
}

func (a *Appender) patchAndAppend(path string, existing []byte, newContent string, now time.Time) error {
  fm := frontmatter(existing)
  patched := updatedAtRe.ReplaceAll(fm,
    []byte("updated_at: "+now.Format("2006-01-02 15:04")))

  // Reconstruct: patched frontmatter + rest of file + new content
  rest := existing[len(fm):]
  full := append(patched, rest...)
  full = append(full, []byte(newContent)...)

  return os.WriteFile(path, full, 0o600)
}

// frontmatter extracts bytes between first --- and second --- (inclusive).
func frontmatter(content []byte) []byte {
  // Find opening ---
  // Find closing ---
  // Return slice including both delimiters
}
```

**Key property**: If `updated_at` is not in frontmatter, `hasUpdatedAt`
returns false and the original `O_APPEND` path is used. Zero behavior
change for files without frontmatter — backward compatible.

### 6.6 12h Time Format

```go
// TimeFormat values.
const (
  TimeFormat24h TimeFormat = "24h"
  TimeFormat12h TimeFormat = "12h"
)

func formatTime(t time.Time, format TimeFormat) string {
  switch format {
  case TimeFormat12h:
    return t.Format("3:04 PM")
  default:
    return t.Format("15:04")
  }
}
```

In `appender.go`, replace:
```go
fmt.Fprintf(&buf, "%s %s\n", SubheadingLevel, display.Format("15:04"))
```
with:
```go
fmt.Fprintf(&buf, "%s %s\n", SubheadingLevel, formatTime(display, a.timeFormat))
```

### 6.7 ParseLastTimestamp — Dual Format

```go
// Match both "## 14:30" and "## 2:30 PM"
var timestampRe = regexp.MustCompile(
  `(?m)^` + SubheadingLevel + ` (\d{1,2}):(\d{2})(?:\s*(AM|PM))?\s*$`,
)

func ParseLastTimestamp(content string) (time.Time, bool) {
  matches := timestampRe.FindAllStringSubmatch(content, -1)
  if len(matches) == 0 {
    return time.Time{}, false
  }

  last := matches[len(matches)-1]
  h, _ := strconv.Atoi(last[1])
  m, _ := strconv.Atoi(last[2])
  ampm := last[3] // "" for 24h, "AM" or "PM" for 12h

  if ampm != "" {
    if ampm == "PM" && h != 12 {
      h += 12
    } else if ampm == "AM" && h == 12 {
      h = 0
    }
  }

  if h > 23 || m > 59 {
    return time.Time{}, false
  }

  return time.Date(0, 1, 1, h, m, 0, 0, time.UTC), true
}
```

This handles both formats transparently. Existing 24h files parse the
same as before. New 12h files parse correctly.

## 7. Design Decisions & Trade-offs

### DEC-001 — Frontmatter patching via line-level regex

**Options**:
- A) Full YAML parse → modify → marshal. Safe but heavy; may reformat YAML.
- B) Line-level regex: find `updated_at: ...` line, replace value. Light,
  preserves formatting. Risk: won't handle multi-line YAML values.

**Decision**: B — line-level regex. `updated_at` is always a simple scalar
(`updated_at: 2026-03-09 14:30`), never multi-line. Regex is simpler, faster,
and preserves the user's file formatting. The frontmatter boundary (between
`---` delimiters) constrains the search scope.

### DEC-002 — New `internal/template` package

Template loading, parsing, and rendering is a distinct concern from appending.
New package keeps the appender focused on its existing responsibilities.
The template package exports `Render(path, Context)` — the appender calls it
for new files only.

### DEC-003 — Append path splits by frontmatter presence

When `updated_at` is absent: original `O_APPEND` path (fast, safe).
When `updated_at` is present: read-modify-write via `os.WriteFile` (needed
to patch frontmatter). The `hasUpdatedAt` check examines only the frontmatter
section, not the entire file. Files created with the default template (no
frontmatter) always use the fast path.

### DEC-004 — Dual regex for timestamp parsing

Single regex `(\d{1,2}):(\d{2})(?:\s*(AM|PM))?` handles both 24h and 12h.
AM/PM capture group is optional — absent means 24h. Conversion to 24h
internal representation in the parser. Existing tests pass without change.

### DEC-005 — `title_format` as Go template

The config value `title_format = "Log — {{ .date }}"` is a Go template
rendered with `{"date": "Monday, 2 January 2006"}`. This is consistent
with the daily file template being Go `text/template`. Alternative
(printf-style) rejected — less flexible, different syntax from the file
template.
