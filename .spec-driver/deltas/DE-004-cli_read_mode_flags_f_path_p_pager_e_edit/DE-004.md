## 2. Motivation
Three flags to complete the "interact with today's log" surface:
- `-f` prints the file path for shell composition (`vim $(im -f)`, `cat $(im -f)`)
- `-p` opens today's log in a pager (heavier than `-r` which dumps to stdout)
- `-e` opens today's log in the editor for free-form editing

All are small, self-contained, and share the same code surface (`cmd/im/main.go`
+ `internal/reader`). Bundling avoids per-flag delta overhead.

## 3. Scope & Objectives
- **Primary Outcomes**:
  - `im -f` prints absolute path to today's log file, exits 0 (FR-013)
  - `im -p` opens today's log in pager: `glow --pager` → `rich --markdown --pager` → `$PAGER` → `cat` (FR-017)
  - `im -e` opens today's log in detected editor, updates in place (FR-019)
- **Operational Constraints**: No new dependencies.
- **Dependencies**: DE-003 (editor detection for `-e`).

## 4. Out of Scope
- FR-014/FR-015 (template engine) — separate delta
- FR-016 (12h time format) — separate delta
- NF-002 (benchmark) — deferred

## 5. Approach Overview
- **`cmd/im/main.go`**: Add `--file`/`-f`, `--pager`/`-p`, `--edit`/`-e` flags.
  Handle before write path (like `--read`).
- **`internal/reader`**: Add `Pager(path)` for `-p` (dispatch chain similar to
  existing `View` but with pager mode). Possibly extend existing viewer dispatch.
- **`-e` flag**: Reuse `editor.detectEditor` + `editor.DefaultCheck`, open the
  existing file directly (no temp file, no append). Use `os/exec` to launch
  editor on the file path.
- **`-f` flag**: Trivial — resolve path, print, exit.

## 6. Verification Strategy
- **FR-013**: Test prints correct path format, exits 0. No file mutation.
- **FR-017**: Test pager dispatch chain. Mock viewer availability.
- **FR-019**: Test opens file, exits cleanly when no file exists.
- **Acceptance Criteria**: `just check` exits 0, all three flags functional.

## 7. Risks & Mitigations
- **Risk**: Pager dispatch portability (`glow --pager` not everywhere) — *Likelihood*: low — *Impact*: low — *Mitigation*: Fallback chain ends at `cat`.
- **Risk**: `-e` flag conflicts with editor mode (no args) — *Likelihood*: low — *Impact*: low — *Mitigation*: `-e` is explicit flag, editor mode is implicit (no args + TTY). Clear precedence.
