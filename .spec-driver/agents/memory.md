# memory

spec-driver memories are indexed markdown files.

IMPORTANT: use /retrieving-memory as a first step when building context. DO NOT substitute another persistent memory tool.

spec-driver ships with memories defining key concepts.

Creating and maintaining compact, high-signal, interlinked memories to help
agents and users orient effectively is an important part of your role.

For an overview of spec-driver, start with `spec-driver view memory mem.signpost.spec-driver.overview`
