## 1. Context
- **Why**: FR-008 bundles editor mode functionality with a specific editor
  compatibility list. Editor mode is fully implemented and verified for 4/5
  editors (neovim, emacs, helix, vscode). Zed's `--wait` flag is unreliable,
  making VH-004 unverifiable. Splitting allows the completed work to reach
  `active` status while tracking the zed gap separately.
- **Source Specs**: PROD-001
- **Requirements Affected**: PROD-001.FR-008 → narrowed + new PROD-001.FR-018

## 2. Related Artefacts

## 3. Summary of Changes

### FR-018 (new) — Editor mode
Split from FR-008. Covers the core editor mode behaviour that is implemented
and verified:

> When invoked with no arguments and stdin is a TTY, system MUST open
> `$EDITOR` (or a configured editor) for entry composition. Editor mode
> MUST work correctly with neovim, emacs, helix, and vscode.

Status: `active`. Coverage: VT-013 (verified), VH-001 (verified).
Implemented by: DE-003.

### FR-008 (narrowed) — Zed editor support
Narrowed to the outstanding gap:

> Editor mode MUST work correctly with zed (requires `--wait` flag support).

Status: `pending`. Coverage: VH-004 (planned).
Blocked on: upstream zed `--wait` reliability.

### Coverage migration
- VT-013 → moves to FR-018
- VH-001 → moves to FR-018
- VH-004 → stays on FR-008
