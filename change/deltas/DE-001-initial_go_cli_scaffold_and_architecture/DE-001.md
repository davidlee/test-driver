## 1. Summary & Context
- **Product Spec**: [PROD-001](../../specify/product/PROD-001/PROD-001.md) – Interstitial Mindfulness Logger
- **Design Revision**: [DR-001](./DR-001.md) – Go project structure and package architecture
- **Implementation Plan**: [IP-001](./IP-001.md) – 3 phases: module/build, config, CLI/input modes

## 2. Motivation
No code exists yet. This delta bootstraps the Go module, establishes the
package layout, config loading, and CLI flag parsing — the skeleton that all
functional deltas will build on.

## 3. Scope & Objectives
- **Primary Outcomes**:
  - Go module initialised with build tooling (`go.mod`, `Makefile` or `justfile`)
  - Package structure defined and documented in a tech spec
  - Config loading from `~/.config/im/im.toml` with sensible defaults (NF-001)
  - CLI flag parsing (`-t`, `-r`, `--`) stubbed
  - Binary compiles as `im` (NF-003)
- **Operational Constraints**: Core dep: urfave/cli. TOML parsing via stdlib or lightweight lib.
- **Dependencies**: None (first delta).

## 4. Out of Scope
- Actual entry capture, file writing, timestamp logic (future deltas)
- Editor integration
- Read mode / pager

## 5. Approach Overview
- **Key Changes**:
  - `cmd/im/main.go` — entrypoint
  - `internal/config/` — TOML loading with defaults
  - `internal/cli/` — flag/arg parsing, input mode detection
  - Tech spec SPEC-001 for the `im` binary architecture

## 6. Verification Strategy
- **Acceptance Criteria**:
  - `go build ./cmd/im` produces a working `im` binary
  - `im --help` prints usage
  - Config loads defaults when no `im.toml` exists
  - Config reads overrides from `im.toml` when present
  - Unit tests for config loading and CLI parsing

## 7. Risks & Mitigations
- **Risk**: Dependency weight — *Likelihood*: low — *Impact*: low — *Mitigation*: urfave/cli is the only significant dep; TOML parsing via `BurntSushi/toml` or similar lightweight lib.

## 8. Follow-ups & Tracking
- **Future Deltas**: Daily file management (FR-001–003), timestamp logic (FR-004, FR-011), input modes (FR-005–010), read mode.
