## 2. Motivation
The only remaining input mode. When `im` is invoked with no args and stdin
is a TTY, it should open `$EDITOR` for multi-line composition. This
completes the input mode matrix. The `editor_timestamp` config key
(`"start"` / `"end"`) controls whether the entry gets the time the editor
was opened or saved.

Reference implementation: user sample code — editor detection, launch,
content validation. We'll simplify for `im`'s narrower needs.

## 3. Scope & Objectives
- **Primary Outcomes**:
  - Editor opens on `im` with no args + TTY (FR-008)
  - Saved content becomes entry body, appended to daily file
  - Empty/aborted edit → no file mutation (consistent with existing empty guard)
  - `editor_timestamp` config: `"start"` (default) or `"end"` (FR-010)
  - Works with neovim, emacs, helix, zed, vscode
- **Operational Constraints**: Single new dep (`github.com/confluentinc/go-editor`
  or equivalent). Must not break existing < 1ms inline path.
- **Dependencies**: DE-002 complete (merged).

## 5. Approach Overview
- **New package**: `internal/editor` — simplified from user sample code
  - Editor detection chain: config override → `$EDITOR` → fallback list
    (nvim, hx, helix, vim, emacs, nano, code --wait)
  - Launch via temp file, return content + exit code
  - Content validation: UTF-8, size limit, binary detection
  - Save logic: exit 0 → save; non-zero → discard (simplified from vice)
- **Modified packages**:
  - `internal/config` — add `editor` field to config (already exists but unused)
  - `cmd/im/main.go` — wire editor mode: detect → open editor → apply
    timestamp strategy → append

## 6. Verification Strategy
- **Unit tests**: editor detection (with injected command checker), save
  decision logic, content validation
- **Integration**: end-to-end with a mock editor script that writes known
  content to the temp file
- **Acceptance Criteria**:
  - `im` (no args, TTY) opens editor, saved text appears in daily file
  - Empty save → no mutation
  - `editor_timestamp = "end"` uses post-editor time
  - `just check` exits 0

## 7. Risks & Mitigations
- **Risk**: Editor flag differences (--wait for vscode/zed) — *Likelihood*: medium — *Impact*: low — *Mitigation*: Detection chain includes flags per editor; document in config.
- **Risk**: TTY detection in CI/test — *Likelihood*: medium — *Impact*: low — *Mitigation*: Existing `cli.IsTerminal` var is already stubbed for tests.
