# DR-003 – Editor mode and timestamp config

## 1. Executive Summary
- **Delta**: [DE-003](./DE-003.md)
- **Status**: draft
- **Synopsis**: Add `internal/editor` package for external editor integration,
  wire into `cmd/im/main.go` for the editor input mode.

## 2. Problem & Constraints
- **Current Behaviour**: `im` with no args + TTY prints a stub message and exits.
- **Drivers**: FR-008 (editor mode), FR-010 (editor timestamp config). Last
  missing input mode.
- **Constraints**: Must not regress inline path performance. Editor must work
  with neovim, emacs, helix, zed, vscode. Single new dependency acceptable.
- **Out of Scope**: Template engine (DE-004), `-f`/`-p` flags, 12h time.

## 3. Architecture Intent
- **Target Outcomes**:
  - `im` (no args, TTY) → opens editor → saved content appended to daily file
  - Empty/aborted → no file mutation
  - `editor_timestamp = "start"` (default): timestamp = time editor opened
  - `editor_timestamp = "end"`: timestamp = time editor closed
- **Guiding Principles**: Crib from vice's `exteditor`, simplify for `im`'s
  narrower needs. Keep the package testable via injected command checker.

## 6. Code Sketches

### `internal/editor` — public API

```go
package editor

// DefaultEditors is the fallback chain when no config or $EDITOR is set.
var DefaultEditors = []string{
  "nvim", "hx", "helix", "vim", "emacs", "nano", "code --wait",
}

// CommandChecker reports whether a command is available on PATH.
type CommandChecker func(cmd string) bool

// Edit opens an external editor for entry composition. Returns the
// edited content. Returns ("", nil) when the user aborts or saves
// empty content — caller should treat this as "no entry".
func Edit(cfg config.Config, check CommandChecker) (string, error)
```

The package is stateless — no struct, no constructor. `CommandChecker` is
the only injection point, defaulting to `exec.LookPath` in production and
a stub in tests.

### Editor detection chain

```go
func detectEditor(cfg config.Config, check CommandChecker) (string, error) {
  // Build candidate list: config → $EDITOR → defaults
  var candidates []string
  if cfg.Editor != "" {
    candidates = append([]string{cfg.Editor}, DefaultEditors...)
  } else if env := os.Getenv("EDITOR"); env != "" {
    candidates = append([]string{env}, DefaultEditors...)
  } else {
    candidates = DefaultEditors
  }

  for _, ed := range candidates {
    bin := strings.Fields(ed)[0]
    if check(bin) {
      return ed, nil
    }
  }
  return "", fmt.Errorf("no editor found; set $EDITOR or install one of: %v",
    DefaultEditors)
}
```

### Editor launch via `go-editor`

```go
import "github.com/confluentinc/go-editor"

func launchEditor(editorCmd string) (string, int, error) {
  ed := editor.NewEditor()
  ed.Command = editorCmd

  content, tmpPath, err := ed.LaunchTempFile("im-*.md", strings.NewReader(""))
  if tmpPath != "" {
    defer os.Remove(tmpPath)
  }

  if err != nil {
    var exitErr *exec.ExitError
    if errors.As(err, &exitErr) {
      return "", exitErr.ExitCode(), nil // non-zero exit, not an error
    }
    return "", 1, fmt.Errorf("editor failed: %w", err)
  }

  return string(content), 0, nil
}
```

The library handles temp file creation, writing initial content, launching
the editor command, and reading back the result. We just need cleanup.

### Save logic (simplified from reference)

```go
func shouldSave(exitCode int, content string) bool {
  return exitCode == 0 && strings.TrimSpace(content) != ""
}
```

Reference code had a secondary branch (non-zero exit + original empty +
new content → save). `im` always starts with empty content, and we want
non-zero exit to mean "abort" unconditionally — simpler mental model.

### Timestamp wiring in `cmd/im/main.go`

The interesting bit: Appender already takes an injected clock. Editor
timestamp config just selects which captured time to feed it.

```go
// Before editor launch
startTime := time.Now()

body, err := editor.Edit(cfg, editor.DefaultCheck)
if err != nil { ... }
if body == "" { os.Exit(0) } // empty guard

endTime := time.Now()

// Select timestamp per config
ts := startTime
if cfg.EditorTimestamp == "end" {
  ts = endTime
}

appender := logfile.NewAppender(cfg, func() time.Time { return ts })
appender.Append(logDir, body, task)
```

No new abstraction needed — just a closure over the chosen time.

### Content validation

```go
func validateContent(content string) error {
  if !utf8.ValidString(content) {
    return fmt.Errorf("content is not valid UTF-8")
  }

  const maxSize = 10 * 1024 * 1024 // 10 MB
  if len(content) > maxSize {
    return fmt.Errorf("content too large: %d bytes (max %d)", len(content), maxSize)
  }

  if containsBinary(content) {
    return fmt.Errorf("content appears to contain binary data")
  }
  return nil
}

func containsBinary(content string) bool {
  if strings.Contains(content, "\x00") {
    return true
  }
  // > 5% non-printable (excluding \t \n \r) → likely binary
  nonPrintable := 0
  for _, r := range content {
    if r < 32 && r != '\t' && r != '\n' && r != '\r' {
      nonPrintable++
    }
  }
  return content != "" && float64(nonPrintable)/float64(len(content)) > 0.05
}
```

Cheap safety net — rejects accidental binary pastes or encoding corruption.

## 7. Design Decisions & Trade-offs

### DEC-001 — Simplify from user sample code

The reference `EditorOpener` takes `context.Context`, has a struct with
injectable checker, and returns content + error with nuanced save logic
(non-zero exit saves if original was empty and new content exists).

For `im`:
- Drop `context.Context` — not needed, no cancellation.
- Flatten to package-level functions — no struct needed, inject checker
  via function param.
- Simplify save logic: exit 0 + non-empty content → save. Everything else
  → discard. `im` always starts with empty content, so vice's "original was
  empty" branch is the only case — but we simplify further: non-zero exit
  means abort, period.
- Keep: content validation (UTF-8, size, binary detection) — cheap safety net.

### DEC-002 — Use `github.com/confluentinc/go-editor`

Same lib as reference code. Handles temp file creation, editor launch, content read-back.
Known to work with neovim, emacs, helix, vscode. Lightweight.

### DEC-003 — Editor timestamp config

Two capture points:
- `"start"` (default): record `time.Now()` before launching editor. The
  timestamp reflects when the thought arose.
- `"end"`: record `time.Now()` after editor returns. The timestamp reflects
  when the entry was committed.

Implementation: `cmd/im/main.go` captures `startTime` before editor call,
`endTime` after. Passes the selected one to `Appender.Append` (which already
accepts an injected clock — pass a closure returning the chosen time).

### DEC-004 — Editor fallback chain

Simplify from reference:
```
config.Editor → $EDITOR → nvim → hx → helix → vim → emacs → nano → code --wait
```

Changes from reference:
- Drop `kak` (niche, not in FR-008 editor list).
- Drop `-w` from `vim` (not a standard flag across vim builds).
- Add `--wait` to `code` (required for vscode to block until close).
- `zed --wait` not in default chain — user can set via config/`$EDITOR`.

## 7. Open Questions
- **YAGNI — stdin TTY mode**: A mode where `im` (no args, TTY) reads from
  stdin until ^D instead of opening an editor. Potentially useful for quick
  multi-line input without editor overhead. Deferred — pipe mode covers most
  of the same ground.
