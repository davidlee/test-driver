# DR-004 – CLI read-mode flags: -f path, -p pager, -e edit

## 1. Executive Summary
- **Delta**: [DE-004](./DE-004.md)
- **Status**: draft
- **Synopsis**: Add `-f`, `-p`, `-e` flags and fix `-r` stdout/pager
  distinction. Completes the "interact with today's log" surface.

## 2. Problem & Constraints
- **Current Behaviour**: Only `-r` (read) exists. It calls `reader.View`
  which dispatches to `glow --pager` — this is pager behavior, not stdout
  rendering per FR-012. No `-f`, `-p`, or `-e` flags exist.
- **Drivers**: FR-013 (-f), FR-017 (-p), FR-019 (-e), FR-012 correction.
- **Constraints**: No new dependencies. `-e` reuses editor detection from
  DE-003. Must not regress write path.
- **Out of Scope**: FR-014/FR-015 (templates), FR-016 (12h time).

## 3. Architecture Intent
- **Target Outcomes**:
  - `im -f` → prints absolute path to today's log, exits 0 (FR-013)
  - `im -r` → renders today's log to stdout via `glow` or `cat`, no pager (FR-012 fix)
  - `im -p` → opens today's log in pager (FR-017)
  - `im -e` → opens today's log in detected editor for in-place editing (FR-019)
  - Empty file cases: `-f` prints path regardless, `-r`/`-p`/`-e` print
    "no entries for today" and exit 0
- **Guiding Principles**: Each flag is a single function in `cmd/im/main.go`.
  Reader package owns viewer dispatch. Editor package owns editor launch.

## 4. Code Impact Summary

| --- | --- | --- |

## 6. Code Sketches

### `-f` flag — print file path (FR-013)

Trivial. Resolve path, print, exit.

```go
func runFile(logDir string) error {
    today := time.Now().Format("2006-01-02") + ".md"
    path := filepath.Join(logDir, today)
    fmt.Println(path)
    return nil
}
```

No file existence check — the path is valid whether or not entries exist
today. Shell composition needs the path regardless (`touch $(im -f)`).

### `-r` fix + `-p` flag — stdout vs pager (FR-012 / FR-017)

Split `reader` dispatch into two functions:

```go
// Render writes the file to stdout (for -r). No pager.
// Dispatch: glow → cat.
func Render(path string) error {
    if _, err := os.Stat(path); os.IsNotExist(err) {
        return ErrNoFile
    }

    bin, argv, err := resolveRenderer(path)
    if err != nil {
        return err
    }

    return syscall.Exec(bin, argv, os.Environ())
}

func resolveRenderer(filePath string) (string, []string, error) {
    candidates := []viewer{
        {name: "glow", args: nil},        // stdout, no --pager
        {name: "cat", args: nil},
    }
    for _, v := range candidates {
        bin, err := exec.LookPath(v.name)
        if err != nil {
            continue
        }
        argv := append([]string{v.name}, v.args...)
        argv = append(argv, filePath)
        return bin, argv, nil
    }
    return "", nil, fmt.Errorf("no viewer found")
}
```

Existing `View` (with `--pager` dispatch) becomes the `-p` handler.
`-r` calls `Render` instead of `View`.

### `-e` flag — edit existing file (FR-019)

New function in `internal/editor`:

```go
// EditFile opens an existing file in the detected editor.
// Unlike Edit, no temp file — the file is edited in place.
func EditFile(path string, cfgEditor string, check CommandChecker) error {
    editorCmd, err := detectEditor(cfgEditor, check)
    if err != nil {
        return fmt.Errorf("detecting editor: %w", err)
    }

    parts := strings.Fields(editorCmd)
    parts = append(parts, path)
    cmd := exec.Command(parts[0], parts[1:]...)
    cmd.Stdin = os.Stdin
    cmd.Stdout = os.Stdout
    cmd.Stderr = os.Stderr
    return cmd.Run()
}
```

No temp file, no content validation, no save/discard logic. The user is
editing their own file directly — we trust them.

Wiring in `cmd/im/main.go`:

```go
func runEdit(cfg config.Config, logDir string) error {
    today := time.Now().Format("2006-01-02") + ".md"
    path := filepath.Join(logDir, today)

    if _, err := os.Stat(path); os.IsNotExist(err) {
        fmt.Println("no entries for today")
        return nil
    }

    return editor.EditFile(path, cfg.Editor, editor.DefaultCheck)
}
```

### Flag dispatch in run()

```go
func run(_ context.Context, cmd *cli.Command, cfg config.Config) error {
    logDir, err := cfg.ResolvedLogDir()
    if err != nil {
        return fmt.Errorf("resolving log dir: %w", err)
    }

    switch {
    case cmd.Bool("file"):
        return runFile(logDir)
    case cmd.Bool("read"):
        return runRead(logDir)
    case cmd.Bool("pager"):
        return runPager(logDir)
    case cmd.Bool("edit"):
        return runEdit(cfg, logDir)
    default:
        return runWrite(cmd, cfg, logDir)
    }
}
```

## 7. Design Decisions & Trade-offs

### DEC-001 — Split reader into Render (stdout) and View (pager)

Current `reader.View` uses `glow --pager` — pager behavior. FR-012
specifies stdout without a pager. Rather than parameterizing `View`,
add `Render` with a separate dispatch chain (`glow` → `cat`, no
`--pager`). `View` stays as-is for `-p`. Cleaner than a mode flag.

### DEC-002 — EditFile uses os/exec directly

`go-editor` library manages temp files — unnecessary for editing an
existing file. Direct `os/exec.Command` with stdin/stdout/stderr
connected is simpler and has no cleanup concerns.

### DEC-003 — Fix -r as part of this delta

Current `-r` calls `reader.View` (pager mode). FR-012 says "no pager".
Since we're adding `-p` for pager, fix `-r` to call `Render` (stdout)
in the same delta. Small scope increase, avoids leaving a known
mismatch.

## 8. Observation — -r behavior mismatch

The current `-r` implementation uses `glow --pager`, which is pager
behavior (FR-017 intent). FR-012 says "render to stdout, no pager".
VT-010 verifies viewer dispatch but doesn't distinguish pager vs
stdout. This delta fixes the mismatch by wiring `-r` to `Render`.
