## 1. Summary & Context
- **Product Spec**: [PROD-001](../../specify/product/PROD-001/PROD-001.md) – Interstitial Mindfulness Logger
- **Depends on**: DE-001 (Go CLI scaffold)

## 2. Motivation
DE-001 built the skeleton. This delta makes `im` actually useful: writing
entries to daily Markdown files with timestamp subheadings, reading them
back, and applying task-checkbox formatting. After this delta, the tool is
usable for all non-editor input modes.

## 3. Scope & Objectives
- **Primary Outcomes**:
  - Daily file creation with date heading (FR-001)
  - Append entries without mutating prior content (FR-002)
  - Configurable log directory with `~/log` default (FR-003)
  - Timestamp subheading emission — adaptive + round10 modes (FR-004, FR-011)
  - Inline args and `--` separator write entries (FR-005, FR-006)
  - Piped stdin writes entries (FR-007)
  - `-t` flag prefixes lines with `- [ ] ` (FR-009)
  - `-r` flag displays today's log via pager/glow/cat (FR-012)
- **Dependencies**: DE-001 complete (merged).

## 4. Out of Scope
- Editor mode (FR-008, FR-010) — deferred to DE-003
- NF-002 performance benchmarking — verify informally, formal bench later

## 5. Approach Overview
- **New packages**:
  - `internal/logfile` — daily file creation, last-timestamp parsing, append
  - `internal/entry` — text composition, task prefix
  - `internal/reader` — pager/viewer dispatch
- **Modified packages**:
  - `internal/cli` — read stdin pipe content
  - `cmd/im` — wire everything together, replace diagnostic stubs

## 6. Verification Strategy
- **Unit tests**: timestamp rounding (both strategies), file creation, append
  logic, task prefix, last-timestamp parsing
- **Integration tests**: end-to-end: `im` writes to a temp dir, verify file
  contents match expected Markdown
- **Acceptance Criteria**:
  - `im hello` creates `YYYY-MM-DD.md` with date heading + timestamp + entry
  - Second `im` within 10m appends under same heading (adaptive mode)
  - `im -t buy milk` produces `- [ ] buy milk`
  - `echo text | im` captures piped input
  - `im -r` displays the file
  - `just check` exits 0

## 7. Risks & Mitigations
- **Risk**: File race on concurrent `im` invocations — *Likelihood*: low — *Impact*: medium — *Mitigation*: Use file-level locking or append-mode open; acceptable for single-user tool.
- **Risk**: Timestamp parsing fragility — *Likelihood*: low — *Impact*: medium — *Mitigation*: Strict regex for `## HH:MM` headings, comprehensive test cases.

## 8. Follow-ups & Tracking
